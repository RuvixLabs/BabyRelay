# BabyRelay web fallback

This static package owns the public caregiver-invite fallback for
`babyrelay.app`.

- Installed iOS/Android apps should intercept `/join/<code>` through Universal
  Links / App Links.
- Browsers validate the six-character invite code and forward it to the
  `babyrelay-meta` AppRefer capture page as `invite_code` for deferred install
  attribution.
- The Apple association file is static and scoped only to `/join/*`.
- Android association data is generated by `api/assetlinks.js`. Set the Vercel
  environment variable `ANDROID_SHA256_CERT_FINGERPRINTS` to one or more
  comma-separated uppercase Play App Signing SHA-256 fingerprints. The route
  returns `503` instead of publishing an unverified association when unset.

The root [`vercel.json`](../vercel.json) supplies the rewrites and JSON content
types. Vercel must serve both `/.well-known/*` paths directly with HTTP `200`;
do not redirect those paths from the apex domain to `www`.

Run the local checks with:

```sh
npm test --prefix website
node --check api/assetlinks.js
node --check website/join.js
```
